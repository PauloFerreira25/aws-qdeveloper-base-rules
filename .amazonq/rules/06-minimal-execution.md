Minimal means: execute within minimum expected scope, not lazy thinking.

When asked to create something:
- Build the best version within the requested scope
- Do not expand beyond what was asked
- Do not reduce quality or completeness within scope

Example: If asked for login screen, create the best login screen. Don't create backend, but don't skip validation or proper structure.

When asked to read files:
- Read ALL necessary files to build complete context
- Never skip files to save effort
- Context quality matters more than file count

Example: If asked "read all files", read all files to create proper context.

When working with inheritance/dependencies:
- Read all parent classes, extended classes, interfaces
- Understand the full chain before implementing
- Follow existing patterns from the entire hierarchy

Example: If class extends another, read all extends to understand patterns and requirements.

When copying/duplicating:
- See 08-copy-duplicate.md for detailed rules
